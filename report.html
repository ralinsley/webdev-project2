How to use

This is a simple database with only one table, and two columns (not including the SERIAL id). First is "secret" which is a TEXT field for a or any random thought. Second is "secret_type" which is VARCHAR(20) - up to 20 characters but preferably one word that describes the general category/type for the secret. And here are example requests to test the endpoints. (Please don't put any actual secrets in here)

Create: 

curl --header 'Content-Type: application/json' \     
            --data '{"secret": "xx xx xx xx x.", "secret_type": "xxxx"}' \
            https://project2.apolity.me:8443/confess

Read: visit url: https://project2.apolity.me:8443/get to return all secrets in the database and https://project2.apolity.me:8443/get/id to return a secret with that "id".

Update: where "id" matches the id of the secret to update. 
curl --HEADER 'Content-Type: application/json' 
     --request PATCH --data 
     '{"secret": "xx xx xx xx x.", "secret_type": "xxxx"}' 
        https://project2.apolity.me:8443/update/id


Delete: curl --request DELETE https://project2.apolity.me:8443/delete/id

  1. It is not a to-do list! It's a service that holds secrets confessed by users. (any random thought)
  2. The web service uses a mysql database
  3. It is an express based web service
  4. available on: https://project2.apolity.me:8443/
  5. /confess to post a new confession/secret. /update/:id to update/change a secret with that id. /get and /get/:type to get all secrets and to get all secrets of a type. /delete/:id delete secret with that id.
  6. When users are posting a new secret, or updating a secret it must receive a JSON payload. The response from the server are held in JSONs.
  7. All CRUD operations are available.
  8. The firewall should be all setup, and direct access to the database require credentials.
  9. Nginx proxy server is setup.
  10. Cross-origin resource sharing should be setup properly following Chapter 12 from allover.
